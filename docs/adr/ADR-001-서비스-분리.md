# ADR-001: 마이크로서비스 분리 전략

## 상태
승인 (Accepted)

## 컨텍스트
컨퍼런스 관리 시스템을 설계할 때 모놀리식 vs 마이크로서비스 아키텍처를 결정해야 한다.
- 3개 도메인: 참석자 관리, 세션 관리, 발표 제안(CFP)
- 각 도메인은 독립적인 비즈니스 로직과 데이터를 가짐
- 팀 간 독립적 배포가 요구됨

## 결정
도메인 경계(Bounded Context)에 따라 3개 마이크로서비스로 분리한다:
- **attendee-service**: 참석자 등록/관리
- **session-service**: 세션 일정/관리
- **cfp-service**: 발표 제안 접수/투표

공유 모델은 `common` 모듈(Shared Kernel)로 관리한다.

## 결과
### 긍정적
- 각 서비스의 독립적 배포/확장 가능
- 도메인별 기술 스택 선택 자유
- 장애 격리 (Blast Radius 축소)

### 부정적
- 서비스 간 통신 복잡성 증가
- 데이터 일관성 관리 어려움
- 운영 오버헤드 증가
